# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**AWS Multi-OU Billing Aggregator** — A pipeline that collects billing data from multiple AWS accounts across Organization Units (OUs) using Cost Explorer API, then aggregates it in Databricks Delta Live Tables (DLT) using a medallion architecture (Bronze → Silver → Gold).

## Architecture

### Data Flow

```
AWS Management Account (Terraform)
  └─ Creates per-account IAM roles: DatabricksBillingRole-{account_id}
     └─ Trust relationship with Databricks instance profile role
        └─ ExternalId: "databricks-billing-aggregator"

Databricks Cluster (with instance profile)
  └─ Assumes each DatabricksBillingRole-{account_id} (STS AssumeRole)
     └─ Calls AWS Cost Explorer API
        └─ Returns daily cost breakdown by service, region, usage type, linked account
           └─ Feeds into DLT pipeline
```

### DLT Table Lineage

| Layer | Table | Purpose |
|-------|-------|---------|
| **Bronze** | `aws_billing_bronze` | Raw Cost Explorer records (append-only, partitioned by OU/account) |
| **Silver** | `aws_billing_silver` | Type conversion, deduplication, quality checks (date casting, year/month extraction) |
| **Gold** | `aws_billing_daily_by_service` | Daily aggregation by OU, account, service, region |
| — | `aws_billing_monthly_by_ou` | Monthly rollup at OU level with account/service counts |
| — | `aws_billing_top_services` | Top services by spend across all OUs |

### Medaillon Layers in `billing_pipeline.py`

- **Bronze**: Reads raw records from `collect_all_billing_records()` — all pagination handled
- **Silver**: Applies `@dlt.expect_or_drop` quality gates, date conversions, dropDuplicates
- **Gold**: 3 separate aggregation views; each reads from Silver

## Key Technical Decisions

### Single Management Account Provider (Terraform)

The Terraform code demonstrates IAM setup using the **management account provider** with `for_each` to iterate over all OU/account pairs. For **production multi-account deployments**, use:
- **Terragrunt** — wrapper for DRY Terraform with per-account support
- **AWS CloudFormation StackSets** — handles stack deployment across accounts
- **Provider aliases per account** — each account gets its own provider block with assumed role

See `terraform/main.tf` lines 131–140 for commented example of per-account provider pattern.

### Error Handling Strategy

- **Driver-side collection** (`collect_all_billing_records()`): If a single OU/account fails to assume role or fetch data, it logs a warning and **continues** (line 204). One failing account should not abort the entire pipeline.
- **DLT expectations** (Silver layer): `@dlt.expect_or_drop` gates reject invalid rows (null amounts, invalid accounts, missing periods). Rejected rows go to quarantine table (auto-generated by DLT).

### Cost Explorer Pagination

- Uses Cost Explorer's `NextPageToken` loop
- Called with `DAILY` granularity
- Groups by SERVICE, REGION, USAGE_TYPE, LINKED_ACCOUNT
- Each call is paginated inside `fetch_cost_and_usage()` (generator pattern)

### Configuration Via Pipeline Parameters

All inputs are injected by DLT via `spark.conf` in `databricks.yml`:
- `billing_role_arns` — JSON map of OU → {account_id: role_arn}
- `query_start` / `query_end` — date range (Cost Explorer max 12 months back)
- `external_id` — must match Terraform trust policy condition
- `target_catalog` / `target_schema` — where tables are created

## Common Commands

### Terraform: IAM Role Setup

```bash
cd terraform

# Initialize (one-time, or after provider version bump)
terraform init

# Preview changes
terraform plan -var-file=terraform.tfvars

# Apply IAM roles and policies
terraform apply -var-file=terraform.tfvars

# Output Role ARNs for Databricks config
terraform output -json billing_role_arns
```

### Adding/Modifying OU Accounts

1. Edit `terraform/terraform.tfvars`:
   ```hcl
   ou_accounts = {
     "production" = ["111111111111", "222222222222"]
     "compliance" = ["666666666666", "777777777777"]  # Add new OU
   }
   ```

2. Plan and apply:
   ```bash
   cd terraform
   terraform plan -var-file=terraform.tfvars  # Shows new roles to create
   terraform apply -var-file=terraform.tfvars
   terraform output -json billing_role_arns    # Copy new ARNs
   ```

3. In Databricks UI or `databricks.yml`, update `billing_role_arns` parameter with new role ARNs

### Databricks: Deploy / Modify DLT Pipeline

1. **Create/Update `databricks.yml`** with pipeline configuration:
   ```yaml
   resources:
     pipelines:
       aws_billing_aggregator:
         libraries:
           - notebook:
               path: /Workspace/databricks/billing_pipeline.py
         configuration:
           pipelines.billing_role_arns: |
             { "production": { "111111111111": "arn:aws:iam::111111111111:role/DatabricksBillingRole-111111111111", ... } }
           pipelines.query_start: "2024-01-01"
           pipelines.query_end: "2024-12-31"
           pipelines.target_catalog: "billing"
           pipelines.target_schema: "aws_costs"
           pipelines.external_id: "databricks-billing-aggregator"
   ```

2. **Deploy via Databricks CLI**:
   ```bash
   databricks bundle deploy
   ```

3. **Run pipeline**:
   ```bash
   databricks bundle run aws_billing_aggregator
   ```

4. **Check logs**:
   - Go to Databricks workspace → Pipelines → aws_billing_aggregator
   - Check "Logs" tab for warnings/errors during Cost Explorer fetch

## Adding New DLT Tables

All DLT tables are defined in `databricks/billing_pipeline.py`. To add a new Gold-layer aggregation:

1. Add a new function decorated with `@dlt.table()`:
   ```python
   @dlt.table(
       name=f"{TARGET_CATALOG}.{TARGET_SCHEMA}.aws_billing_<new_name>",
       comment="Your description",
       table_properties={"quality": "gold"},
   )
   def aws_billing_<new_name>():
       return (
           dlt.read(f"{TARGET_CATALOG}.{TARGET_SCHEMA}.aws_billing_silver")
           .groupBy(...)
           .agg(...)
       )
   ```

2. Commit and deploy via Databricks bundle / UI

3. If new Cost Explorer dimensions are needed (e.g., by tag, cost allocation tag), update **both**:
   - `fetch_cost_and_usage()` — add to `GroupBy` list (line 146–150)
   - `BRONZE_SCHEMA` — add new StructField
   - Silver table — add type conversions if needed

## Troubleshooting Notes

### AssumeRole Failures

Check these in order:
1. **Instance Profile Role ARN** — Ensure `terraform.tfvars` `databricks_role_arn` matches the cluster's actual instance profile
2. **ExternalId mismatch** — Verify both Terraform trust policy (line 125) and DLT config parameter `external_id` use same value
3. **IAM role permissions** — Confirm Databricks cluster role has `sts:AssumeRole` permission to assume `DatabricksBillingRole-*` roles

### Empty or Delayed Cost Data

- Cost Explorer may take 24–48 hours to report previous day's costs
- Query window `query_start` / `query_end` must be within last 12 months
- Use DLT pipeline logs to confirm Cost Explorer returned non-empty result sets

### DLT Expectations Quarantine

If many rows drop during Silver layer:
- Check DLT quarantine table for examples: `dlt.read("<table>_quarantine")`
- Most common: null amounts (service had zero usage that day)
- Adjust or remove `@dlt.expect_or_drop` as needed
